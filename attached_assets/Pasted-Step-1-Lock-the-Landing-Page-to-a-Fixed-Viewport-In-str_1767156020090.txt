Step 1 — Lock the Landing Page to a Fixed Viewport

In streamlit_app.py, inside the landing page render block only:

Wrap the landing page content in a container with a fixed height:

st.markdown("""
<style>
.landing-container {
    height: 100vh;
    overflow: hidden;
    display: flex;
    flex-direction: column;
    justify-content: space-between;
}
</style>
""", unsafe_allow_html=True)


Wrap all landing page elements in:

st.markdown('<div class="landing-container">', unsafe_allow_html=True)
# existing landing page content here
st.markdown('</div>', unsafe_allow_html=True)


Do not apply this CSS globally—scope it to the landing page only.

Step 2 — Reduce White Space Above the Hero Image

Modify the hero image CSS:

Reduce top margin and padding to zero:

.hero {
    margin-top: 0;
    padding-top: 0;
}


If a Streamlit container is adding spacing above the image, override it:

section[data-testid="stVerticalBlock"] > div:first-child {
    padding-top: 0;
}


Confirm the image visually anchors to the top of the viewport without cropping key content.
